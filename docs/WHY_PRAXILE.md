# Why Praxile?

Praxile is a governed experience harness for AI coding. It does not try to replace coding agents, IDE agents, or spec tools.

## The Boundary

| Tool or method | Primary job | Praxile's job |
|---|---|---|
| Claude Code-style agents | Execute coding tasks through a strong operational harness | Record what happened and govern what becomes durable repo experience |
| Spec Kit / SDD tools | Govern intent before coding | Use specs as optional context, then govern experience after execution |
| Hermes-style agents | Grow a general-purpose agent over time | Keep repository-local coding experience scoped, auditable, and reviewable |
| OpenClaw-style gateways | Connect assistants to channels and personal workflows | Keep local code-project experience under `.praxile/` |
| Ordinary RAG memory | Retrieve text that may help a run | Track evidence, proposal review, lifecycle, usage, and negative feedback |
| Markdown skills | Store human-readable instructions | Treat skills as approved assets with source tasks, scope, anti-scope, and rollback |

## The Short Version

Spec-driven development governs what an AI coding agent should build before execution.

Praxile governs what the agent actually learned after execution.
