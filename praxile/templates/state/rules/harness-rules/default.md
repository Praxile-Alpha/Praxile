# Default Harness Rules

## Privacy-sensitive tasks

Prefer local/private model roles for tasks involving secrets, credentials, auth data, customer data, unreleased business logic, or repository-specific confidential context.

## Verification

Run configured tests or explain why tests are unavailable. Do not treat missing tests as the same signal as passing tests.

## Evolution

Memory, skill, eval, failure pattern, project pattern, frozen boundary, and harness rule updates must remain proposals until explicitly accepted by the user.

## Safety

Respect protected paths, dangerous command blocks, sensitive-file filters, diff review, backups, and rollback records.
