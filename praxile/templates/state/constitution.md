# Praxile Experience Constitution

## Immutable Governance Principles

1. No durable asset without evidence.
2. No global rule from a single run.
3. No memory update without scope and anti-scope.
4. No skill accepted if it cannot define when not to use it.
5. No proposal accepted without source task and rollback path.
6. No silent overwrite of accepted experience.
7. Negative feedback must lower confidence or trigger lifecycle review.
8. Architecture-sensitive changes require explicit review.
